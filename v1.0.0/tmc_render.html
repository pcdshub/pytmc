tmc_render.py¶

tmc_render.py

This file contains the tools for recursively exploring the TwinCAT program structure, parsing relevant pragmas and rendering the resulting EPICS db file.

SingleRecordData¶

class pytmc.SingleRecordData(pv=None, rec_type=None, fields=None, comment=None)¶

Data structre for packaging all the information required to render an epics record.

SingleRecordData supports equality testing. Two instances are equal if all their attributes are equal.

Note

The parameters for the constructor are for setting the attributes of this class.

SingleProtoData¶

class pytmc.SingleProtoData(name=None, out_field=None, in_field=None, init=None)¶

Data structre for packaging all the information required to render an epics proto entry.

SingleProtoData supports equality testing. Two instances are equal if all their attributes are equal.

Note

The parameters for the constructor are for setting the attributes of this class.

Parameters

  • name (str) –

  • out_field (str) –

  • in_field (str) –

  • init (str) –

name¶

This specifies the name of the proto ‘method’. This is the name used to refer to the proto in the .db file. Name collision is prevented in make_intf() by manually assigning a name for the from_element_path().

Type

str

out_field¶

This is the content to be placed after the ‘out’ field in the proto file.

Type

str

in_field¶

This is the content to be placed after the ‘in’ field in the proto file.

Type

str

init¶

If this field is meant to be initialized with another value, place the name of the proto here.

Type

str
